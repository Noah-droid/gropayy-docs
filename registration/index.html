User Registration API

Overview

This API endpoint allows users to register by providing their personal details, contact information, and role.

Endpoint

URL: /api/v1/user/registration
Method: POST
Content-Type: application/json
Authentication: Not required for registration

Request Body

The request must be in JSON format with the following fields:

Field Type Required Description
firstName string ✅ Yes User's first name
middleName string ❌ No User's middle name (optional)
lastName string ✅ Yes User's last name
gender string ✅ Yes User's gender (e.g., Male, Female)
phone string ✅ Yes User's phone number (e.g., 2345762296334)
email string ✅ Yes User's email address
password string ✅ Yes User's password (must meet security requirements)
bvn string ❌ No User's Bank Verification Number (optional)
permissions array<string> ✅ Yes List of permissions assigned to the user
superAgentId integer ❌ No ID of the super agent, if applicable
role string ✅ Yes User's role (e.g., "USER")

Sample Request

{
  "firstName": "John",
  "middleName": "Doe",
  "lastName": "Smith",
  "gender": "Male",
  "phone": "2345762296334",
  "email": "john@example.com",
  "password": "SecurePass123!",
  "bvn": "12345678901",
  "permissions": [
    "PERMISSION_ACCOUNT_CREATE"
  ],
  "superAgentId": 10,
  "role": "USER"
}

Response

The API returns a JSON response indicating the success or failure of the registration.

Success Response

Status Code: 201 Created

{
  "step": "STEP_1",
  "nextStep": "STEP_1",
  "userId": 0,
  "email": "string",
  "fullName": "string",
  "createdDate": "string",
  "updatedDate": "string",
  "role": "string",
  "message": "string"
}

Error Responses

Status Code Error Message Description
400 Bad Request Invalid input Some required fields are missing or incorrectly formatted
409 Conflict User already exists The email or phone number is already in use
500 Internal Server Error Server error An unexpected error occurred

Notes

  • Ensure that the email and phone number are unique.
  • Password should be strong and follow security best practices.
  • If superAgentId is provided, it must be a valid reference.
